# データ取得・加工仕様書

本プロジェクトにおける、5ちゃんねる「ニュース速報+」掲示板からのデータ取得および解析の仕様をまとめます。

## 1. データソース
- **URL**: 環境変数 `SOURCE_URL` から取得します。
- **デフォルト値**: `https://asahi.5ch.io/newsplus/subject.txt`
- **文字コード**: `Shift_JIS` (デコードして処理)

## 2. 解析・フィルタリング処理
取得した `subject.txt` の各行に対して以下の処理を行います。

### データのパース
各行を `.dat<>` をセパレーターとして分割し、左側から「スレ立て日時のUnix Time」、右側から「タイトル（コメント数・投稿者名・シリーズ数を含む）」を抽出します。

### フィルタリング
以下の条件に当てはまる行は無視（除外）します。
- **古い/特殊なスレッド**: Unix Timeの先頭が「9」で始まるもの。
- **コメント不足**: タイトル末尾のカッコ内にあるコメント数が50未満のもの。
- **不正なシリーズ**: 「★」があるがその直後に数字がないもの。

### メタデータの抽出
- **スレ立て日時**: ファイル名（Unix Time）から復元。
- **コメント数**: タイトル末尾の `(数字)` から抽出。
- **投稿者名**: タイトル内の `[投稿者名]` から抽出（※UI上は現在非表示）。
- **シリーズ数**: タイトル内の `★数字` から抽出。未指定の場合は「初代(1)」として扱う。

## 3. 「勢い」の計算
「勢い（res/d）」は、スレ立て時点から現在までの平均的なコメント増加率を24時間換算したものです。

**計算式**:
$$勢い = \frac{コメント数}{経過時間(時間)} \times 24$$
※経過時間（時間） = $(\text{ビルド時の現在時刻} - \text{スレ立て時刻}) \div 3600$

## 4. 重複排除とランキング
- **同一シリーズの統合**: 「★」を除いたベースタイトルが同じスレッドが複数見つかった場合、**最も勢い（数値）が高い1つだけを採用**し、他は無視します。
- **ソート**: 勢いの降順（数値が大きい順）に並べ替えます。
- **抽出件数**: 上位**20件**を抽出します。

## 5. UIレイアウト
画面解像度 800x480 をターゲットとした、1行表示の超シンプルデザインです。

- **表示方向**: 縦リスト形式
- **行内レイアウト**: 
    - **左側**: 勢い数値 ＋ タイトル（溢れた分は「...」で省略）
    - **右側**: 日時（`MM/DD hh:mm` 形式）
- **配色**: 背景白、文字黒、勢い数値は赤系の強調色
- **補足**: 番号（順位）やアイコン、投稿者、コメント数は極限まで省略したデータ特化型
